## Purpose

This document defines the naming conventions for tables, columns, indexes, constraints, and schemas.

## Standards

- Use lowercase snake_case for database objects
- Use descriptive names aligned to business concepts
- Prefer singular names for entities and plural names for collections where appropriate
- Use consistent prefixes for audit and metadata columns
